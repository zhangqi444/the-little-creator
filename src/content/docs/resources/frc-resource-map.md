A curated map of where to actually go for FIRST Robotics Competition information. Every link below is annotated with what it is, who it's for, and when to use it. We don't republish content — we point at it. Official sources are authoritative; independent and community resources are flagged accordingly. Our FRC coverage is newer and lighter than our FLL coverage — additions welcome via PR.

## Official sources (canonical)

### FIRST Robotics Competition — program home

- **URL:** https://www.firstinspires.org/programs/frc/
- **Authority:** official
- **Audience:** families, coaches, teachers, teens
- **Level:** all
- **Tags:** frc, official, program-home
- **Use when:** You need program-wide info on FRC — what it is, who it's for, current season highlights, and entry points to deeper resources.

The official FRC program landing page on FIRST Inspires. Covers ages 14–18 / grades 9–12. Current season: REBUILT presented by Haas (2025/26, released January 2026). **Next season: BIOCORE presented by Haas, releasing January 9, 2027**, under the FIRST CANOPY umbrella. Stats from 2024-25 show worldwide participation across hundreds of regional events.

### Get Started with FIRST Robotics Competition

- **URL:** https://www.firstinspires.org/programs/frc/get-started
- **Authority:** official
- **Audience:** families, teachers, coaches
- **Level:** beginner
- **Tags:** frc, official, getting-started, registration
- **Use when:** You are starting a new FRC team — need the canonical onboarding flow including team size, coach requirements, materials, and cost expectations.

The FRC-specific getting-started flow on FIRST. Walks through who you need on the team (typically 10–20+ students grades 9–12, plus 2 required lead adult coaches; many teams add professional engineering mentors), what hardware and software is required, and routes you to registration. Note: FRC has a substantial financial commitment — typical first-year cost is $5,000+ before custom fabrication.

### REBUILT — current FRC season (2025/26)

- **URL:** https://www.firstinspires.org/programs/frc/game-and-season
- **Authority:** official
- **Audience:** coaches, teams, families
- **Level:** all
- **Tags:** frc, season-2025-26, rebuilt, current

The current FRC season hub on FIRST. **REBUILT presented by Haas** is the 2025/26 theme — teams use engineering skills to re-imagine the past, score fuel, cross obstacles, and climb the tower. Page links to the official game animation, a five-part field tour video series, KitBot resources, and downloadable gameplay cards. URL rotates to the next season landing each January.

### FRC KitBot — beginner-friendly robot starting point

- **URL:** https://www.firstinspires.org/resources/library/frc/kitbot
- **Authority:** official
- **Audience:** coaches, rookie teams, programmers
- **Level:** beginner
- **Tags:** frc, kitbot, beginner, hardware
- **Use when:** You are a new FRC team building your first robot — the KitBot provides a versatile, beginner-friendly starting point included in the Kickoff Kit.

The FRC KitBot is the canonical starter-robot kit included in the FIRST Robotics Competition Kickoff Kit. Page provides build instructions, CAD files, programming guides, and enhancement tips to help rookie teams get competition-ready. Past KitBot resources are archived season-by-season for teams referencing prior years' designs.

### Newly Formed FRC Teams (Rookie criteria)

- **URL:** https://www.firstinspires.org/resources/library/frc/new-team-criteria
- **Authority:** official
- **Audience:** new coaches, organizers
- **Level:** beginner
- **Tags:** frc, rookie, new-team, eligibility, official
- **Use when:** You are forming a new FRC team and need to know whether you qualify for Rookie Status — affects rookie awards and championship eligibility.

The official criteria for FRC Rookie Status. A team qualifies as Rookie if (1) it's associated with a school or organization that did not have an FRC team in the last 3 seasons, AND (2) does not have 6 or more student participants with prior FRC competition experience. Petitions for Rookie Status under edge-case circumstances go to customerservice@firstinspires.org with "Request for 2026 Rookie Status" in the subject line.

### WPILib — FRC robotics library and tools

- **URL:** https://wpilib.org/
- **Authority:** official
- **Audience:** teens, coaches
- **Level:** intermediate, advanced
- **Tags:** frc, programming, wpilib, java, cpp, open-source
- **Use when:** You need the canonical FRC programming library and tooling — install VS Code with WPILib extensions, find API docs, browse worked examples for Java and C++.

WPILib is the canonical robot programming library for FRC, supporting Java, C++, and (historically) LabVIEW. Site provides installer, documentation, API reference, and worked examples. Used by the vast majority of FRC teams. Maintained jointly by FIRST and a community of contributors; releases are coordinated with the FRC season cycle.

### WPILib documentation — long-form reference

- **URL:** https://docs.wpilib.org/en/stable/
- **Authority:** official
- **Audience:** teens, coaches, mentors
- **Level:** beginner, intermediate, advanced
- **Tags:** frc, wpilib, documentation, java, cpp, python, official
- **Use when:** Looking up a specific WPILib API, learning the command-based framework, troubleshooting a roboRIO deployment, or following step-by-step zero-to-robot tutorials.

The full long-form WPILib documentation hub — separate from the short marketing pages at wpilib.org. Hosted on Read the Docs. Covers installation, the Zero-To-Robot tutorial sequence, command-based programming (the recommended pattern for new teams), telemetry and dashboards, motor controllers, sensors, simulation, and advanced topics (state-space control, system identification). Now ships official Python support (RobotPy) alongside Java and C++. Treat this as the canonical reference; the marketing site exists to point here.

## Hardware vendors

FRC teams source from a small ecosystem of specialist vendors. Most teams use a mix — REV or CTRE for motor controllers, AndyMark for wheels and structural parts, REV ION or West Coast Products for chassis. These entries cover the dominant vendors.

### REV Robotics — FRC ION product line

- **URL:** https://www.revrobotics.com/frc/
- **Authority:** independent (long-time FRC vendor)
- **Audience:** coaches, mentors, teens
- **Level:** intermediate, advanced
- **Tags:** frc, hardware, rev-ion, brushless, control-system, kits
- **Use when:** Sourcing brushless motors (NEO, NEO Vortex), SPARK MAX / SPARK Flex motor controllers, the REV ION mechanical extrusion system, or the official FRC Starter Bot for rookie teams.

REV's FRC product line is branded **REV ION** — note it's distinct from the FTC-side **REV DUO**, even though both are made by the same company. Major products: the NEO and NEO Vortex brushless motors, SPARK MAX and SPARK Flex motor controllers (industry-standard for non-Falcon teams), MAXTube and MAXPlanetary mechanical systems, and the **REV ION FRC Starter Bot** — a buildable beginner robot REV ships each season aligned to the game. ION docs at [docs.revrobotics.com/docs/ion](https://docs.revrobotics.com/docs/ion).

### CTRE — Cross The Road Electronics

- **URL:** https://www.ctr-electronics.com/
- **Authority:** independent (long-time FRC vendor)
- **Audience:** mentors, teens, advanced programmers
- **Level:** intermediate, advanced
- **Tags:** frc, hardware, ctre, motors, motor-controllers, swerve, falcon, kraken
- **Use when:** Sourcing the Talon SRX / Talon FX / Kraken X60 / Kraken X44 motor controllers and motors, CANcoders (absolute encoders), Pigeon IMU, or building a swerve drivetrain using a SwerveModuleConstants-backed library.

CTRE's Phoenix ecosystem is one of two dominant motor-controller stacks in FRC (REV being the other). Their **Kraken** brushless motors with integrated FOC control are widely used on competitive swerve drivetrains. The Phoenix 6 software framework is the canonical CTRE programming interface — well-integrated with WPILib's command-based pattern. Documentation at [pro.docs.ctr-electronics.com](https://pro.docs.ctr-electronics.com/).

### AndyMark — long-time FRC mechanical vendor

- **URL:** https://www.andymark.com/
- **Authority:** independent (long-time FRC vendor)
- **Audience:** mentors, coaches, teens
- **Level:** beginner, intermediate
- **Tags:** frc, hardware, andymark, mechanical, gearboxes, wheels
- **Use when:** Sourcing wheels (Colson, Mecanum, pneumatic), gearboxes (Toughbox, Sport, Evo), structural parts, electrical components, or other FRC-legal mechanical hardware not made by REV or CTRE.

AndyMark has been an FRC vendor since the program's early years. Carries a broad catalog of FRC-legal parts — wheels, gearboxes (including the Toughbox family widely used in rookie and intermediate teams), shaft hardware, batteries, electrical, and structural components. Often the choice for "the part REV/CTRE doesn't make" or for legacy designs. Strong rookie-team voucher participation through FIRST Choice.

### FIRST Choice — team voucher store

- **URL:** https://firstchoicebyandymark.com/
- **Authority:** official (run via AndyMark for FIRST)
- **Audience:** coaches, mentors, rookie teams
- **Level:** beginner
- **Tags:** frc, official, firstchoice, vouchers, kit-of-parts, kickoff
- **Use when:** Spending the team's seasonal FIRST Choice credits on vendor products — every registered FRC team gets a voucher allocation as part of registration.

FIRST Choice is the official voucher-based vendor store for FRC teams. Each registered team gets a yearly credit allocation as part of the Kickoff Kit; credits can be redeemed for specific vendor products listed on the store. Rounds and inventory rotate; check early in the season. Operated by AndyMark on FIRST's behalf — accessible via the team Dashboard.

## Community: data, scouting, and discussion

### The Blue Alliance — FRC match and team data

- **URL:** https://www.thebluealliance.com/
- **Authority:** community (widely-cited, FIRST-recognized)
- **Audience:** teens, mentors, scouts, coaches, families
- **Level:** all
- **Tags:** frc, community, scouting, data, statistics, api
- **Use when:** Looking up a team's history, an event's results, match video, team-specific stats, or wiring scouting tools / dashboards through TBA's public API.

The Blue Alliance (TBA) is the canonical community-run FRC data site. Every event, team, match, award, and (where available) match video is searchable. Used by scouting teams to evaluate alliance picks during regional events and by alumni to track their team's lineage. Free public API (with key registration) widely used by team-built scouting apps and pit-display dashboards. Maintained by volunteer engineers, often FRC alumni; donations support hosting.

### Statbotics — predictive analytics for FRC

- **URL:** https://www.statbotics.io/
- **Authority:** community
- **Audience:** scouts, advanced teens, mentors
- **Level:** intermediate, advanced
- **Tags:** frc, community, statistics, scouting, predictions, elo
- **Use when:** Looking at predictive match results, team Elo trends across seasons, or evaluating an event's strength of schedule.

Statbotics provides predictive analytics on top of TBA's raw data — team Elo ratings, match win-probability forecasts, and event-strength metrics. Useful for advanced scouting and for understanding how a team's competitiveness has shifted over multiple seasons. Free and open-source.

### Chief Delphi — canonical FRC forum

- **URL:** https://www.chiefdelphi.com/
- **Authority:** community
- **Audience:** teens, mentors, coaches, parents
- **Level:** all
- **Tags:** frc, community, forum, foundational
- **Use when:** Following game discussion after Kickoff, asking technical questions about a specific subsystem, reading robot reveals, or following major rules clarifications.

Chief Delphi is the largest and longest-running FRC community forum. Also hosts active sub-categories for FTC, FLL, outreach, and rules / Game Q&A reposts.

## Open-source community libraries

WPILib provides the foundation, but most modern competitive teams layer open-source community libraries on top of it. The entries below cover the most-cited libraries — what each one solves and when a team should consider adopting it. All are free and live on GitHub.

### PathPlanner — autonomous path planning and following

- **URL:** https://pathplanner.dev/
- **Authority:** community (widely-adopted)
- **Audience:** teen programmers, mentors
- **Level:** intermediate, advanced
- **Tags:** frc, autonomous, path-planning, swerve, oss
- **Use when:** Designing autonomous routines that move the robot smoothly through a sequence of waypoints — especially on a swerve drivetrain. Almost universal in modern competitive FRC.

PathPlanner is the dominant FRC autonomous path-planning tool. Ships with a GUI editor (PathPlanner Lib) where teams sketch trajectories on a field image, plus runtime libraries for Java, C++, and Python that follow those trajectories at competition time. Integrates with both REV and CTRE drivetrains. Has largely replaced the WPILib trajectory APIs as the default for new teams. Maintained by mjansen4857; sponsored by FRC team 3015.

### Choreo — trajectory optimisation

- **URL:** https://sleipnirgroup.github.io/Choreo/
- **Authority:** community (rapidly-growing)
- **Audience:** advanced teen programmers, mentors
- **Level:** advanced
- **Tags:** frc, autonomous, trajectory-optimization, swerve, oss
- **Use when:** A team wants time-optimal trajectories (rather than the cubic-spline trajectories PathPlanner generates) — useful in tight, fast autonomous routines on a swerve.

Choreo is a newer trajectory optimiser using nonlinear programming to find time-optimal paths within drivetrain kinematics constraints. Steeper learning curve than PathPlanner; pays off in elimination matches where every autonomous second matters. Often used alongside PathPlanner (Choreo for the demanding paths, PathPlanner for the rest). Maintained by Sleipnir Group.

### PhotonVision — vision processing for coprocessors

- **URL:** https://photonvision.org/
- **Authority:** community (widely-adopted)
- **Audience:** advanced teen programmers, mentors
- **Level:** intermediate, advanced
- **Tags:** frc, vision, apriltags, coprocessor, oss
- **Use when:** A team is adding camera-based localisation or game-piece detection to their robot — typically running on a Raspberry Pi, Orange Pi, or similar coprocessor.

PhotonVision is the canonical open-source vision pipeline for FRC. Runs as a standalone app on a Raspberry Pi (or other Linux coprocessor) and streams detected AprilTags, retro-reflective targets, and contour-based game piece poses back to the roboRIO over NetworkTables. Community alternative to commercial vision systems. Common pairing with Choreo / PathPlanner for vision-corrected autonomous.

### AdvantageKit — logging and replay framework

- **URL:** https://docs.advantagescope.org/getting-started/manage-data/log-with-advantagekit/
- **Authority:** community (team 6328 Mechanical Advantage)
- **Audience:** advanced teen programmers, mentors
- **Level:** advanced
- **Tags:** frc, logging, replay, debugging, oss
- **Use when:** A team wants to log every input and output during a match and be able to replay the match deterministically off-robot for debugging.

AdvantageKit (from FRC 6328) is a logging framework that records every sensor input, motor command, and NetworkTable value during a match. The killer feature is **deterministic replay** — you can re-run the match code off-robot in simulation, with the recorded inputs, and reproduce exactly what happened. Indispensable for debugging "it only fails at the regional" intermittent bugs. Pairs with AdvantageScope for visualization.

### AdvantageScope — match data visualizer

- **URL:** https://docs.advantagescope.org/
- **Authority:** community (team 6328)
- **Audience:** teens, mentors, scouts
- **Level:** intermediate, advanced
- **Tags:** frc, visualization, debugging, telemetry, oss
- **Use when:** Visualising log data from a match — robot trajectory on a field map, mechanism states over time, sensor traces. Works with AdvantageKit logs, WPILib data logs, and live NetworkTables.

AdvantageScope is the visualisation front-end paired with AdvantageKit (but also works standalone with WPILib data logs). Plays back logs with synchronised 3D field view, mechanism state, and graph traces. Replaces older tools like Shuffleboard for post-match analysis on serious teams. Maintained by FRC team 6328.

### YAGSL — Yet Another Generic Swerve Library

- **URL:** https://github.com/BroncBotz3481/YAGSL
- **Authority:** community
- **Audience:** advanced teen programmers, mentors
- **Level:** advanced
- **Tags:** frc, swerve, drivetrain, oss
- **Use when:** A team is building a swerve drivetrain from heterogeneous parts (mixing REV and CTRE, or a non-standard module) and wants a generic library that abstracts the motor controller details.

YAGSL is a generic swerve drivetrain library that works across motor controller stacks (REV, CTRE, off-the-shelf SDS modules, custom modules). The intended use is "configure your swerve via JSON and inherit a working SwerveDrive subsystem". Particularly useful for teams whose swerve module choice differs from the most common turnkey solutions. Maintained by BroncBotz3481.
