Every robotics team needs someone responsible for each critical function. The titles don't matter — the coverage does. If no one owns robot inspection prep, it won't happen. If no one owns the notebook, it won't happen. This page maps the roles across all six programs so you can assign coverage intentionally.

---

## How to use this guide

1. **Pick your program** — jump to that section's table.
2. **Check your team against the role list** — is every row owned by someone?
3. **Note the minimum column** — with a very small team, one person can double up on marked roles; don't leave any row blank.
4. **Revisit at the start of each season** — people graduate, leave, or change interests.

For how to develop students into these roles over time, see [Developing Student Leaders](/guides/student-leadership/).

## FLL (FIRST LEGO League — all divisions)

Applies to FLL Discover, FLL Explore, and FLL Challenge. Adjust depth expectations for younger divisions.

| Role | What they own | Min team size to split |
|---|---|---|
| **Head Coach** | Season calendar, parent comms, overall wellbeing | — (always 1 adult) |
| **Assistant Coach** | Backup adult, safety, logistics, scrimmage logistics | ≥ 2 adults total |
| **Drive Team** (FLL Challenge) | Robot Game drivers (max 2 students); attachment swaps | 2 students |
| **Robot Build Lead** | Drive base integrity, attachment storage, repair kit | 5+ students |
| **Programming Lead** | Mission programs, tuning, backup programs on competition day | 5+ students |
| **Innovation Project Lead** | Research, solution design, presentation delivery | 5+ students |
| **Core Values Lead** | Team building activities, poster/display prep, debrief facilitation | 5+ students |
| **Notebook Keeper** | Meeting notes, design decisions, test data logged every session | any size |
| **Equipment Manager** | Kit inventory, battery charging, packing checklist | any size |

**Minimum viable:** 2 students + 1 coach can run FLL Explore. FLL Challenge works best with 4–8 students so each pillar has coverage.

## FTC (FIRST Tech Challenge)

| Role | What they own | Min team size to split |
|---|---|---|
| **Head Mentor** | Season calendar, registration, safety compliance, rule updates | — (always 1 adult) |
| **Student Team Captain** | Meeting agenda, sub-team coordination, judge interview lead | any size |
| **Robot Lead (Mechanical)** | Drivetrain, mechanisms, build quality, inspection compliance | 6+ students |
| **Robot Lead (Electrical)** | Wiring harness, REV Hub config, power distribution, cable management | 6+ students |
| **Software Lead** | TeleOp, Autonomous, tuning, Driver Station setup | 6+ students |
| **Drive Team Captain** | Driver 1 (TeleOp driver), match strategy calls | any size |
| **Driver 2 / Operator** | Mechanism operator during TeleOp | any size |
| **Portfolio Lead** | Engineering Portfolio — updated after every meeting, not at the end | 5+ students |
| **Connect Lead** | Outreach partners, community events, documentation trail | 5+ students |
| **Notebook Keeper** | Session logs feeding the Portfolio; daily engineering records | any size |
| **Pit Crew Lead** | Pit setup, battery rotation, tool kit, match-ready checklist | 5+ students |
| **Scouting Lead** | Match watching, data collection, alliance selection input | 8+ students |

**Minimum viable:** 4 students can cover FTC — pair Robot/Software, pair Portfolio/Connect — but quality in each area will be thin. 8–12 is the comfortable range.

## FRC (FIRST Robotics Competition)

FRC teams are larger (20–50 students is common). Roles split into sub-teams; each sub-team has a Student Lead and a Mentor Advisor.

| Sub-team | Student Lead owns | Mentor Advisor owns |
|---|---|---|
| **Mechanical** | Design CAD, fabrication, assembly, inspection compliance | Safety, tool training, quality review |
| **Electrical** | Wiring, PDH/PDP, CAN bus layout, battery rotation | Compliance checks, crimping sign-off |
| **Programming** | Robot code (WPILib), vision, Dashboard, Autonomous routines | Code review, architecture guidance |
| **Strategy & Scouting** | Match data, pit scouting, alliance selection recommendations | Dataset tool setup |
| **Impact** | Impact essay, presentation, community outreach log, judge interview | Narrative coaching, institutional memory |
| **Drive Team** | Driver, Operator, Human Player, Coach (4–5 roles, student-filled except Coach) | Drive coach (may be adult mentor) |
| **Pit Crew** | Match-ready checklist, robot hand-off protocol, spare parts management | Parts procurement |
| **Media / Outreach** | Social media, sponsor updates, event photography | Approval gate |
| **Leadership Council** | Cross-sub-team coordination, conflict resolution, Captain role at events | Head Mentor oversight |

**Note:** FRC's 6-week build season compresses role ramp-up. Assign leads in September so they enter Kickoff knowing their responsibilities.

## VEX IQ (VIQRC)

VEX IQ teams are typically 2–8 students (ages 8–14). Roles are lighter-weight — the goal is that every student owns something.

| Role | What they own | Notes |
|---|---|---|
| **Coach** | Season plan, registration, adult safety presence | 1 adult required |
| **Driver** | Teamwork Challenge — Controller drive during match | 1–2 students |
| **Operator** | Second Controller (if used); Teamwork partner coordination | 1 student |
| **Builder** | Robot construction, snap-connector integrity, hardware repairs | 1–2 students |
| **Programmer** | VEXcode IQ Blocks or Python, Autonomous Coding Skills program | 1–2 students |
| **Notebook Keeper** | Engineering notebook — meeting logs, design decisions, test results | 1 student (rotate with small teams) |
| **Pit Captain** | Battery charging, spare parts kit, tools, robot transport | 1 student |

**Minimum viable:** 2 students + 1 coach. Split: one student drives/builds, one programs/notebooks. Scale roles as team grows.

## VEX V5 / VRC (VEX Robotics Competition)

| Role | What they own | Min team size to split |
|---|---|---|
| **Head Coach / Advisor** | Registration, rule compliance, mentor support | — (1 adult) |
| **Team Captain** | Meeting structure, sub-team coordination, judge interview lead | any size |
| **Mechanical Lead** | Robot structure, mechanism design, repair kit, inspection | 5+ students |
| **Electrical Lead** | Motor wiring, Smart Port assignments, V5 Brain config | 5+ students |
| **Programming Lead** | VEXcode Pro / PROS TeleOp + Autonomous, sensor tuning | 5+ students |
| **Drive Team** | Driver (Controller 1), Operator (Controller 2) — two distinct roles | 2 students |
| **Notebook Lead** | Engineering notebook — process narrative, test data, design iterations | any size |
| **Strategy Lead** | Match scoring analysis, autonomous win-point planning, scouting | 5+ students |
| **Pit Manager** | Battery rotation, spare Smart Motors, tools, robot-at-a-glance card | 5+ students |

**Minimum viable:** 3 students can cover V5RC — one mechanical, one software, one shared drive/notebook — but depth suffers. 5–8 is the comfortable range.

## VEX U (VURC — collegiate)

VEX U allows up to 15 members per team and fields **two robots**. Roles mirror V5RC but double up on the robot-side.

| Role | What they own |
|---|---|
| **Team Lead** | Season plan, registration, VURC rules, budget, team comms |
| **Robot A Lead (15-inch)** | Design, build, code, and match strategy for the smaller robot |
| **Robot B Lead (24-inch)** | Design, build, code, and match strategy for the larger robot |
| **Mechanical Engineers (×2–4)** | CAD, fabrication, assembly for both robots (one per robot is minimum) |
| **Software Engineers (×2–4)** | PROS C++ codebase for both robots; two-robot autonomous coordination |
| **Electrical Engineers (×1–2)** | Wiring both robots; Smart Port and CAN bus layout |
| **Driver 1 (Robot A)** | Primary driver of the 15-inch robot during competition |
| **Driver 2 (Robot B)** | Primary driver of the 24-inch robot during competition |
| **Notebook Lead** | Engineering notebook — college-level engineering depth expected |
| **Strategy Lead** | Match scouting, four-robot alliance planning, autonomous coordination |
| **Pit Manager** | Pit logistics for two robots simultaneously; battery rotation ×2 |
| **Outreach / Club Admin** | VURC club status at university, funding requests, succession planning |

**Minimum viable:** 4 students (two per robot for build/code) + 2 drivers. Dedicated notebook and strategy roles emerge with 6+.

## Role assignment tips (all programs)

**Name names, not groups.** "The build team owns inspection" fails. "Marcus owns inspection" works.

**Every role needs a backup.** Competition day illness happens. For every critical competition-day role, name a backup in September.

**Rotate roles year to year.** The student who was Notebook Keeper this season becomes Robot Lead next season. Prevents single points of failure and builds broader capability.

**Roles are responsibilities, not hierarchy.** A student can own the notebook without being "in charge." Flat ownership beats ranked authority for most robotics teams.

**Check coverage quarterly.** At the start of the season, mid-season, and before the competition, do a five-minute "who owns what" check. Uncovered roles surface early enough to fix.
