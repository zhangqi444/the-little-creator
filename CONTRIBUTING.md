# Contributing to The Little Creator

Thank you for wanting to help! This community grows through the contributions of families like yours.

## Ways to Contribute

### 1. Fix or Improve Existing Content
Found a typo? Outdated link? Better way to explain something? Just edit the file and submit a pull request.

### 2. Add a Resource
Know a great book, video, or tool that helped your team? Add it to the relevant page in `docs/resources/` and submit a PR.

### 3. Share Your Team's Story
Submit your team to the [Showcase](docs/showcase/README.md). Use the template provided and create a folder under `docs/showcase/[year]-[team-name]/`.

### 4. Write a Guide
Have hard-won experience to share? Write it up as a markdown file in `docs/guides/` and submit a PR. Guides don't need to be polished — honest and useful is what matters.

### 5. Open a Discussion
Not ready to write a full guide? Start a conversation in [GitHub Discussions](../../discussions). Your experience helps others even in casual form.

## How to Submit a Pull Request

If you're new to GitHub, here's the short version:

1. **Fork** this repository (button at top right of the repo page)
2. Make your changes in your fork
3. Submit a **Pull Request** back to this repo
4. We'll review and merge it — usually within a few days

If you've never done this before, [GitHub's guide](https://docs.github.com/en/get-started/quickstart/contributing-to-projects) walks through it step by step.

## Content Guidelines

- Write for families — assume the reader is a parent or a kid, not an engineer
- Be specific and practical — "we did X and it worked because Y" is more useful than general advice
- Keep it honest — sharing what didn't work is just as valuable as sharing what did
- No promotion — this is a community space, not a place to sell products or services

## Questions?

Open a Discussion or an Issue and we'll help you out.
